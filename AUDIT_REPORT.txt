CRITICAL BUGS (3)
=================

BUG 1: engine.py:309 - CRITICAL - Wrong Cost Calculation
Problem: cost_per_share = order.fill_price + (Config.SLIPPAGE_BPS / 10000) + (Config.FEE_RATE)
Issue: Adds BPS instead of multiplying. If fill_price=$100, SLIPPAGE_BPS=2:
  Current: 100 + 0.0002 + 0 = 100.0002
  Should be: 100 * (1 + 0.0002 + 0) = 100.02
Impact: P&L calculations completely wrong, position sizing too aggressive
Fix: cost_per_share = order.fill_price * (1 + (Config.SLIPPAGE_BPS / 10000) + Config.FEE_RATE)


BUG 2: backtester.py:84-105 - CRITICAL - Wrong P&L Units
Problem: pnl = (close - entry_price) * 100 / entry_price (percentage)
         total_pnl += pnl (accumulates percentage, not currency)
Issue: Results show "500% total return" instead of "$5,000 P&L"
Impact: All backtest metrics in wrong units, strategy evaluation corrupted
Fix: Remove * 100 from line 84


BUG 3: engine.py:72 - CRITICAL - Timezone Mismatch
Problem: Uses naive datetime.utcnow() for filtering database queries
Issue: May not match timezone-aware datetimes in database
Impact: Daily trade counting may be off, circuit breaker timing wrong
Fix: Standardize on naive UTC datetimes throughout codebase


MAJOR BUGS (3)
